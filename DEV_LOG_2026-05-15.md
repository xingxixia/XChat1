## 1. 启动流程调整

原来的启动逻辑是：


这样程序一启动，聊天框和立绘会同时出现。

```cpp

// dialogWin.show();

```

文件：

`E:\File\AIuseing\xai\ZcChat2\main.cpp`

这样启动后只显示立绘，更接近桌宠体验。

注意：

- `Dialog dialogWin;` 不能删。
- 因为立绘右键打开聊天框依赖已有信号槽：

```cpp
QObject::connect(&tachieWin, &Tachie::requestToggleVisible, &dialogWin,
                 &Dialog::ToggleVisible);
```

也就是说，聊天框对象仍然存在，只是启动时不显示。

## 3. 设置窗口越界问题

用户发现：

当对话框比较高，或者靠近屏幕上边缘时，打开设置窗口会出现：

- 设置窗口顶部超出屏幕。
- 标题栏和关闭按钮不可见。
- 窗口层级较高，导致不好关闭，也不好操作其它窗口。

后续确认：

原作者发布的软件也能复现这个问题，不是本地二次开发引入的。

可疑代码：

`main.cpp`

```cpp
settings = new MainWindow(&dialogWin, &tachieWin, &dialogWin);
```

第三个参数 `&dialogWin` 是设置窗口的 parent。

而 Dialog 本身在 `windows/dialog/dialog.cpp` 中设置了：

```cpp
Qt::FramelessWindowHint | Qt::Tool | Qt::WindowStaysOnTopHint
```

推测：

设置窗口可能受到对话框 parent、位置或置顶属性影响，没有被限制在屏幕可用区域内。

## 6. 产品方向理解

今天讨论后，对这个项目的定位更清楚了。

它不是要做一个复杂的 AI 工具平台，而更像是：

```text
Galgame 风格桌面角色 + AI 对话
```

核心体验应该是：

- 桌面上有一个角色。
- 角色能陪用户聊天。
- 对话框像 Galgame。
- 角色有固定人设和语气。
- 可以顺便帮用户解释代码、学习、陪伴。

所以后续二开方向不应该一开始堆很多功能，而是先把最核心的体验做好：

1. 人物好看。
2. 聊天框舒服。
3. 回复像角色。
4. 启动和窗口交互自然。
5. 基础配置稳定。

## 7. API Key 配置后不回话

用户配置了 API Key，但角色不回话。

今天做了初步排查。

配置状态：

`C:\Users\xingxixia\Documents\ZcChat2\config.json`

- DeepSeek API Key 已存在。
- 未打印 key 原文。
- 只确认 key 长度为 35。

当前角色配置：

```json
{
    "modelSelect": "deepseek-v4-flash",
    "serverSelect": "DeepSeek",
    "tachieSize": "20"
}
```

当前选择角色：

```ini
[character]
CharSelect=test
```

发现一个高概率问题：

`Dialog::ReloadAIConfig()` 原本只刷新：

- serviceType
- model

但没有根据当前服务商重新读取 API Key。

原来的构造函数里有硬编码逻辑：

```cpp
QString apiKey = config.value("llm/DeepSeek/ApiKey").toString();
ai->setApiKey(apiKey); // Todo: 根据不同服务类型设置不同的Key
```

这会导致：

- 如果程序启动后才在设置页填 API Key，已创建的 `Dialog` 可能还拿着旧 key。
- 如果切换 OpenAI/DeepSeek，key 也可能不匹配。

今天最后临时修改了：

`E:\File\AIuseing\xai\ZcChat2\windows\dialog\dialog.cpp`

```cpp
void Dialog::ReloadAIConfig()
{
    /*AI初始化*/
    ZcJsonLib CharConfig(ReadCharacterUserConfigPath());
    //读取当前角色的服务商选择
    QString serverSelect = CharConfig.value("serverSelect").toString();
    if (serverSelect == "DeepSeek")
        ai->setServiceType(AiProvider::DeepSeek);
    else if (serverSelect == "OpenAI")
        ai->setServiceType(AiProvider::OpenAI);
    else
    {
        serverSelect = "DeepSeek";
        ai->setServiceType(AiProvider::DeepSeek);
    }
    ZcJsonLib config(JsonSettingPath);
    ai->setApiKey(config.value("llm/" + serverSelect + "/ApiKey").toString());
    //读取当前角色的模型选择
    QString modelSelect = CharConfig.value("modelSelect").toString();
    ai->setModel(modelSelect);

    loadContextHistory();
}
```

注意：

这个修改还没有重新构建验证。

# XChat1 第二版开发日志 - 2026-05-15

## 第二版目标

第二版继续沿着“单个桌面陪伴角色”的方向收束功能。

这一版不再把软件做成多角色导入导出的配置平台，而是先把当前这个桌宠个体的基础体验、提示词配置和历史对话维护做好。

## 1. 聊天框隐藏时同步隐藏历史窗口

问题：

打开聊天框后上滑会显示历史对话窗口，但关闭聊天框时，历史窗口仍然停留在桌面上。用户必须重新打开聊天框再关闭历史窗口。

原因：

历史窗口是独立窗口。原来的聊天框隐藏逻辑只隐藏 `Dialog`，没有同步隐藏 `historyWin`。

修改：

- 增加 `Dialog::hideHistoryWindow()`。
- 在聊天框 `hideEvent()`、`ToggleVisible()` 和 Esc 隐藏逻辑中统一调用。
- 增加 `Dialog::ReloadContextHistory()`，方便设置页修改历史后通知聊天框重新加载。

涉及文件：

```text
windows/dialog/dialog.h
windows/dialog/dialog.cpp
```

## 2. 暴露桌宠行为提示词

问题：

原本 AI 能知道“自己是桌宠”，是因为 `Dialog.cpp` 里硬编码了系统提示词。

这不方便用户在设置界面中调整桌宠的说话风格、陪伴方式和行为定位。

修改：

- 在角色设置页动态增加“桌宠行为设定”文本框。
- 新增 `petPrompt` 字段，保存在当前个体的资源配置中。
- AI 调用时优先读取 `petPrompt`，为空时使用默认桌宠提示词。
- 输出格式约束仍然保留在代码中，避免用户误改后破坏立绘切换和语音解析。

涉及文件：

```text
windows/setting/child/settingchild_char.h
windows/setting/child/settingchild_char.cpp
windows/dialog/dialog.cpp
res/default_config/ZcChat2/Character/Assets/test/config.json
```

## 3. 收束为单个桌宠个体

产品方向：

XChat1 暂时不做多角色管理，而是先把“一个固定桌宠个体”的体验做好。

修改：

- 隐藏角色选择、导入、导出、删除入口。
- 固定当前个体目录为 `test`，继续复用现有资产和用户配置结构。
- 删除 `SettingChild_Char` 中角色导入、导出、删除槽函数。
- 删除导入/导出使用的压缩、解压辅助逻辑。
- 保留立绘大小、模型选择、语音合成、立绘动作绑定等对单个桌宠仍有用的配置。

涉及文件：

```text
windows/setting/child/settingchild_char.h
windows/setting/child/settingchild_char.cpp
```

## 4. 增加历史对话维护页

问题：

原来的历史对话只是追加保存到 `context.json` 的 `history` 数组中。

缺点：

- 错误对话不方便删除。
- API 测试时产生的垃圾上下文会持续污染后续回复。
- 只能在聊天框中上滑查看和回滚，不适合手动维护。

修改：

- 在角色设置底部增加页码切换。
- 第 1 页保留原有设置内容。
- 第 2 页显示历史对话维护界面。
- 支持刷新历史。
- 支持清空全部历史。
- 支持编辑单条历史并保存。
- 支持删除单条历史。
- 支持删除一轮对话，即尽量按“用户一句 + 角色一句”删除。
- 支持回滚到指定历史条目。
- 保存后会发出 `requestReloadContextHistory()`，通知聊天框重新加载内存中的历史。

暂时没有做：

- 长期记忆字段。
- 自动总结。
- 限制发送给模型的历史数量。

这些功能后续等记忆系统设计清楚后再做。

涉及文件：

```text
windows/setting/child/settingchild_char.h
windows/setting/child/settingchild_char.cpp
windows/setting/setting.cpp
windows/dialog/dialog.h
windows/dialog/dialog.cpp
```

## 5. 优化历史维护页可读性

问题：

历史维护页一开始每条记录都是一样的视觉样式，只靠文本前缀里的“用户/角色”区分，阅读时必须细看。

修改：

- 用户行左侧显示 `你`，使用淡蓝色背景。
- 角色行左侧显示 `她`，使用淡粉色背景。
- 异常记录显示 `记录`，使用淡灰色背景。

后续又修正了字体和控件宽度问题：

- 不再使用过窄的固定宽度。
- 根据当前字体实际宽度计算最小宽度。
- 避免 `你`、`她`、`清空全部`、`删一轮` 等文字超出按钮或标签框。

涉及文件：

```text
windows/setting/child/settingchild_char.cpp
```
